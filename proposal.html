The Challenge
What's Holding
Great Service Back
Sanadige's food consistently earns 5-star reviews. But a recurring theme in guest feedback reveals friction points in the experience around the meal — things that don't show up in the kitchen but do show up on TripAdvisor.
📅
Reservation Honoring
Guests report arriving with a confirmed booking only to experience extended waits or seating delays. Trust built at booking is lost at the door.
"Had a reservation but still had to wait 20 minutes to be seated." — TripAdvisor
📋
Menu Availability Confusion
Staff frequently have to inform seated guests that a specific catch or special dish is unavailable — after anticipation has already been built. This is a trust-breaker at a seafood restaurant.
"Ordered a fish dish only to be told it was unavailable." — Google Review
⏱️
Service Speed & Staff Load
Manual order-taking, paper KOTs, and verbal communication between floor and kitchen create bottlenecks, especially during peak service when staff are stretched.
"Good food but service was slow during busy hours." — TripAdvisor
💬
No Guest Communication System
Reservations come in via phone or walk-in, with no structured confirmation, reminder, or pre-arrival communication. Guests forget, arrive at wrong times, or cancel via unanswered calls.
📊
No Operational Visibility
Managers lack a live view of the floor: how many covers tonight, which tables have ordered, what's pending in the kitchen, how revenue tracks against previous weeks.
The Solution
Three Integrated
Systems, One Platform
A custom-built digital stack designed specifically for Sanadige — not an off-the-shelf POS or a generic restaurant app, but a tightly integrated set of tools built around how your team actually works.
1

Every step connected.

Cart & Place Order
Live Catch Availability
The catch strip at the top updates in real-time. When a fish sells out, it shows instantly — no more waiters apologising at the table for unavailable items. Guests browse knowing exactly what's available right now.
AI Dish Concierge
Every dish has an "Ask →" button. Guests can ask anything: "Is this boneless?", "How spicy is this?", "What's the best catch today?", "Is there a vegetarian option?" — answered instantly by Claude AI, trained on your exact menu.
Orders Direct to Kitchen
When a guest taps "Place Order", it goes directly to the kitchen's live board — zero relay through a waiter. Waiters are freed from order-taking to focus on table experience and upselling.
No App Install · PWA
Guests open the menu in their phone's browser by scanning the QR code. It feels like an app — smooth transitions, offline capability, installable to home screen — but requires zero download. Works on all phones.
The QR Code
Each table gets a custom QR code printed on a card or etched into a standee — encoded with the table ID (T-3, Terrace-1, etc.). Scanning opens menu.sanadige.in/T-3 — the menu knows which table placed the order automatically.

Technical Overview
Built to Last,
Not Just to Launch
The entire platform runs on battle-tested infrastructure. No vendor lock-in, no black-box systems, no per-transaction fees from a third-party POS. You own your data.
How It's Built
Dashboard
Next.js web app. Deployed on Vercel. Runs in any browser — tablet, laptop, phone. Instant global delivery, zero downtime updates.
Menu PWA
Separate Next.js app at menu.sanadige.in. Loads in under 2 seconds on 4G. Feels like a native app. Works offline with cached menu.
Backend
Node.js API server on your existing AWS EC2. Handles orders, menu, and AI chat. All logic is yours — no third-party dependencies for core flows.
Database
Supabase (Postgres). Realtime subscriptions for live order updates. Row-level security for staff roles. Fully managed, automatic backups.
AI (Claude)
Anthropic Claude API for the dish chat feature. Menu context is cached for speed. Responses are accurate, safe, and scoped only to Sanadige's menu.
